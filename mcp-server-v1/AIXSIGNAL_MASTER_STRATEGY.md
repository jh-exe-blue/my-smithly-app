| 항목 | 내용 |
|------|------|
| **시작 시점** | 2018년 (기관 전용 트레이딩 데스크) |
| **알파 전략** | 700+ 검증된 퀀트 전략 |
| **AI 노드** | 100+ 오케스트레이티드 AI 분석 |
| **최종 목표** | 금융 민주화 - 기관급 정보를 개인에게 |

### 🌟 최상위 비전: 트리플 컨버전스

```
[전략 전문가 🧠] + [데이터 AI 🤖] + [오픈 커뮤니티 🌐]
          ↓
    신뢰 가능한 판단 흐름 제공
```

**핵심 철학**: 이 세 축의 일치(Convergence)를 통해 신뢰할 수 있는 투자 판단을 제공합니다.

### 🎤 미션 & 슬로건

> **"움직일 때 필요한 건, 단 하나의 신호"**

**미션**: 암호화폐 투자 정보의 격차를 기술과 전문가로 메워, **"알 수 있게 하고 할 수 있게 한다."**

### 💎 핵심 가치 제안 (UVP)

#### 01. 즉시 행동 (Immediate Action)
- 복잡한 분석 없이 바로 움직일 수 있는 **하나의 신호** 제공
- "지금 뭘 해야 하나?"에 대한 명확한 답

#### 02. 납득 가능 근거 (Understandable Evidence)
- 모든 신호에 **X-차트™** 기반의 정량적 근거 제시
- "왜 그래야 하는가?"에 대한 시각적 증명

#### 03. 이해 가능 배경 (Clear Context)
- AI Analyst와 소셜 분석을 통한 명확한 맥락 해석
- "시장에서 무슨 일이 일어나고 있나?"에 대한 설명

## 3. 제품 전략 & 흐름

### 🔄 핵심 사용자 흐름 (UI Flow)

```
1. 행동 (Action) 🚀
   "무엇을 할지"
   ONE 시그널 / 테마
          ↓
2. 근거 (Evidence) 📊
   "왜 해야 하는지"
   X-차트 / AI 분석
          ↓
3. 검증 (Verify) ✅
   "과거엔 어땠는지"
   수익 인증 / 백테스팅
```

### 🔀 듀얼 채널 전략

#### A채널: 전문가 퀀트 축 (단기 실행)

| 속성 | 내용 |
|------|------|
| **대상** | 단기 트레이더 |
| **시간** | 단기 (분/10분봉) |
| **핵심 기능** | ONE 시그널™, X-차트™ |
| **KPI** | 정확도 (Precision) |

**특징**: Execution 중심, 즉각적인 매매 타이밍 제공

#### B채널: AI 내러티브 축 (중기 방향성)

| 속성 | 내용 |
|------|------|
| **대상** | 스윙/일반 투자자 |
| **시간** | 중기 (일/주) |
| **핵심 기능** | i-테마 투자™, 소셜분석 |
| **KPI** | 공감도 (Consensus) |

**특징**: Direction 중심, 시장 흐름과 내러티브 파악

### ✨ 컨버전스 존 (Convergence Zone)

> A채널과 B채널의 신호가 일치하는 **"신뢰도 강화 구간"**  
> UI 최상단에 우선적으로 노출됩니다.

**노출 조건**: 
- A채널 ONE 시그널 발생
- B채널 테마 방향과 일치
- → "지금이 진입 타이밍"이라는 강력한 신호

## 4. 페이지 구조 (IA)

### 📐 공통 PageHeader 구조 (필수)

모든 페이지는 다음 구조를 따라야 합니다:

```
┌─────────────────────────────────────────────────┐
│ [Breadcrumb]                                    │
│ [Title]                        [Primary Action] │
│ [Subtitle / Description]    [Secondary Actions] │
│ [Meta (태그/숫자요약)]                          │
│ [Tabs (페이지 뷰 전환용)]                       │
└─────────────────────────────────────────────────┘
```

### ✅ PageHeader 포함 대상 (Context)

- 페이지 전체 제목 및 설명
- 페이지 레벨의 주요 액션 (저장, 내보내기)
- 핵심 요약 메타 데이터
- 뷰 전환을 위한 탭

### ❌ PageHeader 금지 대상 (Controls)

다음 요소들은 **본문 상단 별도 Bar**로 분리:

- 데이터 필터링 (Filters)
- 검색창 (Search)
- 정렬 순서 변경 (Sort)
- 차트 내부 컨트롤

**이유**: PageHeader는 "무엇을 보고 있는지"만 알려주고, "어떻게 조작할지"는 본문 영역에서 처리

## 6. 기술 아키텍처 및 제약

### ⛔ 핵심 제약 (CRITICAL - 절대 준수)

1. **기존 Socket.IO 설정/핸들러는 절대 수정 금지**
   - 이유: 700+ 전략 서버와의 실시간 연결 안정성 보장
   - 변경 필요 시 반드시 백엔드 팀과 사전 협의

2. **프런트엔드(Lovable)는 WebSocket 구독 전용 (Read-Only)**
   - 데이터 변형 금지: 받은 데이터를 그대로 렌더링
   - 비즈니스 로직은 Supabase Edge Function에서 처리
   - 프런트는 순수하게 UI/UX만 담당

### 🔄 데이터 흐름 (Data Flow)

```
[Strategy Server]    →  [Supabase Edge]    →  [Frontend]
  700+ 전략              처리/AI/저장           WS 구독/렌더링
    (Webhook)              (Realtime)
```

1. **Strategy Server** → Webhook으로 신호 전송
2. **Supabase Edge Function** → AI 분석 + DB 저장 + Realtime 발행
3. **Frontend** → Realtime 구독하여 UI 업데이트

### 🛠️ 기술 스택

#### Chart Technology
- **라이브러리**: Highcharts Stock
- **데이터 형식**: 10분봉, 3종 상태 (trend_long, trend_short, volatility)

#### Backend
- **플랫폼**: Supabase
- **주요 기능**:
  - Edge Functions (AI 처리, 전략 계산)
  - Cron Jobs (정기 업데이트)
  - Realtime (WebSocket 대체)

#### State Management
- **라이브러리**: Zustand
- **원칙**: 불변성(Immutability) 유지 필수

```tsx
// ❌ 잘못된 상태 변경
set((state) => {
  state.signals.push(newSignal); // 직접 변경
});

// ✅ 올바른 상태 변경
set((state) => ({
  signals: [...state.signals, newSignal] // 새 배열 생성
}));
```

## 7. AI 커스터마이징

> AiXSignal의 AI는 단순한 챗봇이 아닌, **특정 작업을 수행하는 전문 분석가**입니다.

### 🤖 작업 1: X-차트™ - AI 시그널 해석가 (MVP)

**목표**: 차트 상태를 초보자에게 쉽게 설명

**Input (JSON)**:
```json
{
  "trend_long": "up",
  "trend_short": "down",
  "volatility": "High"
}
```

**AI Output Example**:
```
"현재 장기 추세는 상승 중이지만, 단기적으로는 가격이 하락하며 조정을 받고 있습니다. 
특히 변동성이 '높음' 상태이므로, 가격이 크게 흔들릴 수 있으니 
신규 진입에 주의가 필요합니다."
```

**Prompt Template**:
```
당신은 암호화폐 차트를 쉽게 설명하는 전문 분석가입니다.
다음 데이터를 40-50대 초보 투자자가 이해할 수 있게 3-4 문장으로 설명하세요:

- 장기 추세: {trend_long}
- 단기 추세: {trend_short}
- 변동성: {volatility}

조건:
- 전문 용어 최소화
- "~입니다" 형식의 공손한 말투
- 구체적인 행동 제안은 피하고, 시장 상태만 설명
```

### 🤖 작업 2: A-뉴스/소셜분석™ - 임팩트 카드 (MVP)

**목표**: 뉴스/소셜 데이터를 분석하여 구조화된 정보 생성

**Input**: Raw 뉴스 텍스트 또는 트위터 스레드

**Output (JSON)**:
```json
{
  "title": "비트코인 ETF 승인 임박",
  "impact": "high",
  "sentiment": "bullish",
  "affected_assets": ["BTC", "ETH"],
  "tldr": "SEC의 ETF 승인이 다음 주로 예상되며, 기관 자금 유입 가능성이 높습니다.",
  "advice": "단기적으로 변동성이 커질 수 있으니, 분할 매수를 권장합니다."
}
```

**Prompt Template**:
```
당신은 암호화폐 뉴스를 분석하는 전문가입니다.
다음 뉴스를 읽고 JSON 형식으로 요약하세요:

{news_content}

반드시 다음 필드를 포함하세요:
- title: 헤드라인 (15자 이내)
- impact: low/medium/high
- sentiment: bullish/neutral/bearish
- affected_assets: 영향 받는 코인 (티커 기호 배열)
- tldr: 한 문장 요약
- advice: 초보 투자자를 위한 행동 조언
```

### 🤖 작업 4: ONE 시그널™ - 신호 통합 엔진 (Phase 2)

**목표**: A채널과 B채널의 신호를 하나로 통합

**로직**:
```typescript
function generateONESignal(aSignal, bSignal) {
  if (aSignal.direction === bSignal.direction && 
      aSignal.confidence > 0.7 && 
      bSignal.consensus > 0.6) {
    return {
      action: aSignal.direction, // "BUY" or "SELL"
      confidence: (aSignal.confidence + bSignal.consensus) / 2,
      zone: "convergence",
      priority: "high"
    };
  }
  return null; // 컨버전스 조건 미달
}
```
